Install the essentials

Two tools make the agent workflow possible. Install them once, use them everywhere.

git — version control

Every change is a commit with a message. Agents can read the history, revert mistakes, and collaborate without stepping on each other. An .xlsx in a git repo is a binary blob that can't be diffed. A .csv or .py file shows exactly what changed, line by line.

# macOS
brew install git

# Windows
winget install --id Git.Git

# Linux
sudo apt install git # Debian/Ubuntu

uv — Python project manager

uv replaces pip, venv, and pip-tools with a single fast tool. It manages Python versions, dependencies, and virtual environments. Agents love it because one command sets up a reproducible environment — no more "it works on my machine."

# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

Your first agent-friendly project

Here's the 60-second version. Open a terminal and follow along:

# 1. Create a project directory
mkdir sales-analysis && cd sales-analysis

# 2. Initialize git (agents love version history)
git init

# 3. Initialize a Python project with uv
uv init

# 4. Add pandas (or polars, or duckdb)
uv add pandas

# 5. Drop in your CSV (the data layer)
cp ~/Downloads/sales.csv data/

# 6. Write your analysis (the logic layer)
echo 'import pandas as pd
df = pd.read_csv("data/sales.csv")
print(df.groupby("region").revenue.sum())
' > analysis.py

# 7. Run it
uv run analysis.py

# 8. Commit (agents can read the history)
git add -A && git commit -m "Initial analysis"

That's it. You now have a project an agent can read, understand, and extend. The CSV is data. The Python file is logic. Git is the audit trail. Export an .xlsx at the end if you need to email it to someone — but the source of truth is this directory.

Data files
